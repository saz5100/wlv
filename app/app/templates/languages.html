🔤 Programming Languages and IDEs

Topic 2.5 covers the differences between high-level and low-level languages, how translators (compilers, interpreters, assemblers) work, and the features of Integrated Development Environments (IDEs).

🎧 Audio Revision: Programming Languages & IDEs
📊 High-Level vs Low-Level Languages ▼
Feature High-Level Language Low-Level Language (Assembly)
Readability Easy — English-like syntax (IF, FOR, PRINT) Hard — Mnemonics (MOV, ADD, LOAD)
Portability Portable — runs on different hardware Hardware-specific — tied to one CPU type
Lines of code 1 line = many machine code instructions 1 line = 1 machine instruction
Speed Slower — needs translation first Faster — closer to machine code
Ease of debugging Easier — meaningful error messages Harder — low-level errors
Examples Python, Java, C#, JavaScript Assembly, LMC (Little Man Computer)
Translator needed Compiler or Interpreter Assembler
🧠 Mnemonic: High-level = Human-friendly but needs translation. Low-level = Machine-friendly, hardware-specific, but faster.
🔄 Compiler vs Interpreter vs Assembler ▼
Feature Compiler Interpreter Assembler
Input High-level code (e.g. Python, Java) High-level code (e.g. Python, JavaScript) Assembly code
Method Translates ALL code at once Executes code line by line Translates assembly to machine code
Output Executable file (.exe) No file — runs directly Object code / machine code
Error reporting Shows ALL errors after compilation Stops at FIRST error Shows syntax errors
Execution speed Fast — already translated Slower — translates each time Fast
Ease of debugging Harder — all errors at once Easier — stops at first error Hard
Used for Production software, apps Scripting, web development Hardware drivers, embedded systems
Examples GCC (C), javac (Java) Python, Node.js, browser JS NASM, MASM
🧠 Mnemonic: Compiler = converts all at once. Interpreter = interprets line by line. Assembler = assembly to machine code.
🛠️ IDE Features ▼

An Integrated Development Environment (IDE) combines all the tools a programmer needs in one application. Know these features for the exam:

✏️

Editor

Write and edit code with syntax highlighting, line numbers, and auto-indentation to make code easier to read.

🔍

Error Diagnostics

Highlights syntax errors as you type — underlines mistakes in red so you can fix them immediately.

🐛

Debugger

Step through code line by line, set breakpoints to pause execution, and inspect variable values at any point.

⏸️

Breakpoints

Mark specific lines where execution should pause, allowing you to examine the program state at that moment.

👁️

Variable Watch

Display the current values of selected variables in real-time as the program executes — essential for finding logic errors.

📚

Call Stack

Shows which subprograms (functions/procedures) are currently active and how they were called — useful for tracing program flow.

💡

Auto-Complete

Suggests code completions, variable names, and function parameters as you type — speeds up coding and reduces typos.

📁

File Management

Browse project files, organise code into folders, and manage multiple source files within the same project.

❓ Common Exam Questions ▼
Question Answer
Why use a high-level language? Easier to read, write, debug. Portable across different hardware. More productive for complex programs.
Why use a low-level language? Faster execution, direct hardware control, smaller memory footprint. Used for device drivers and embedded systems.
Advantage of a compiler over an interpreter? Faster execution after compilation, produces standalone executable, source code not needed at runtime.
Advantage of an interpreter over a compiler? Easier to debug (stops at first error), no need to recompile after changes, better for rapid development.
What is an assembler used for? Translating assembly language (low-level) into machine code. One assembly instruction = one machine instruction.
How does a debugger help? Lets you step through code, set breakpoints, inspect variable values, and trace the call stack to find logic errors.
What is syntax highlighting? An IDE feature that colours different parts of code (keywords, strings, comments) to make it easier to read and spot errors.

📖 Key Terminology for This Topic

These are the exam-grade terms for language techniques. Make sure you can define and use each one.

Simile
A comparison using 'like' or 'as' to create vivid imagery. Example: 'The moon hung like a silver coin in the sky.'
Metaphor
A direct comparison stating one thing is another. Example: 'The city was a concrete jungle.'
Personification
Giving human qualities to non-human things. Example: 'The wind whispered through the trees.'
Alliteration
Repetition of initial consonant sounds. Example: 'The wild, wet wind whipped through the woods.'
Onomatopoeia
Words that imitate sounds. Example: 'The thunder rumbled and the rain pattered on the roof.'
Sensory Language
Language that appeals to the five senses (sight, sound, smell, taste, touch) to create a vivid experience for the reader.
Pathetic Fallacy
Using weather or the environment to reflect a character's emotions. Example: 'The grey clouds mirrored his sorrow.'
Rhetorical Question
A question asked for effect, not expecting an answer. Example: 'Who could ever forget such a sight?'
Repetition
Repeating a word or phrase for emphasis. Example: 'Never give up, never surrender, never stop trying.'
Hyperbole
Exaggeration for dramatic effect. Example: 'I've told you a million times.'
Juxtaposition
Placing two contrasting ideas or images close together for effect. Example: 'The bustling city and the silent countryside.'
Foreshadowing
A hint or clue about what will happen later in the text. Example: 'Little did she know that this was the last time she would see him.'

1. Which of the following is an advantage of a high-level language over a low-level language?
✅ Correct! High-level languages use English-like syntax, making them easier to read, write, and debug. They are also portable across different hardware.
2. A compiler translates source code into machine code. How does it differ from an interpreter?
✅ Correct! A compiler translates all the source code at once into machine code and produces a standalone executable file (.exe).
3. Which translator is used to convert assembly language into machine code?
✅ Correct! An assembler translates assembly language (low-level) into machine code. One assembly instruction equals one machine instruction.
4. Which IDE feature lets you pause execution at a specific line to inspect variable values?
✅ Correct! A breakpoint marks a specific line where execution pauses, allowing you to inspect variables, the call stack, and program state.

✅ Correct! An interpreter stops at the first error, making it easier to debug. It also lets you test changes without recompiling.

✅ Correct! IDEs provide editors, debuggers, breakpoints, variable watches, and call stacks — but they do not automatically optimise your code.
